# hub.reindex

1. Run: `python scripts/hub_index.py` (writes all generated files).
2. Run: `python scripts/hub_lint.py`.
3. If lint reports ERRORs: fix the named files (schemas live in
   /conventions/type-vocabulary.md and /conventions/knowledge-entries.md),
   then re-run both commands until 0 errors.
4. Report: files rewritten, remaining warnings (broken links are allowed).
5. Commit: stage only the regenerated files, NEVER `git add -A` (shared
   checkout, see fact-concurrent-session-git-hygiene):
   `git add memory/index.md features/index.md "features/*/index.md"
   "features/*/knowledge/index.md" narrative/index.md
   narrative/knowledge/index.md views/`, check `git diff --cached --stat`,
   then `git commit -m "chore: reindex"` — but if unrelated changes are
   staged/dirty, ask the user first.

Never hand-edit generated files — they carry the marker
`<!-- generated by scripts/hub_index.py — do not hand-edit -->`.
