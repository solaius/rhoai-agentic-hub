# Agent catalog competitive — hyperscaler moves, analyst view, RH positioning

**Bottom line:** In the 12 months to July 2026, every major platform vendor shipped both an agent *catalog* (curated templates/samples) and an agent *marketplace or registry* (vetted, deployable, governed) — and most are already at GA. The two-tier split RHOAI is sequencing (3.5 link-out templates → 3.6 deployable supported harnesses → MLflow Agent Registry, governed by MCP Gateway/Registry) is exactly the pattern the market converged on, which validates the design but confirms a roughly 12–18 month lag behind first movers on the catalog tier and a larger gap on marketplace/ecosystem scale. The defensible wedge is not "we have a catalog too" — it is *the only open-source, infra-priced, on-prem/air-gapped-capable agent platform with governance built in*, a claim no cloud-only competitor can make and that IBM (the closest analog) makes *on top of* OpenShift rather than instead of it. Analysts (Forrester, Gartner) say governance and observability — not catalog breadth — are the deciding buying criteria, which plays to the MCP Gateway/Registry design. But note one eroding assumption: Google now runs Gemini GA on air-gapped Google Distributed Cloud, so "hyperscalers can't do disconnected" is no longer safely true.

## Google — Agent Garden, Agent Marketplace, and the Vertex AI absorption

Google's move is the most structurally significant of the year: on April 22, 2026 it launched the [Gemini Enterprise Agent Platform](https://cloud.google.com/blog/products/ai-machine-learning/introducing-gemini-enterprise-agent-platform) (Cloud Next '26) and stated that **all Vertex AI services and roadmap will now be delivered exclusively through the Agent Platform** — agents are now the organizing frame of the #2 AI cloud, not a feature of an ML platform ([Virtualization Review, Apr 24, 2026](https://virtualizationreview.com/articles/2026/04/24/google-cloud-next-26-gemini-enterprise-agent-platform-leads-ai-centric-news.aspx)). The platform bundles an Agent Registry (single source of truth for internal agents/tools/skills), Agent Gateway (fleet-wide policy), Agent Runtime/Sandbox, and Security Command Center scanning.

- **Agent Garden** — the direct analog to the RHOAI 3.5 Agent Catalog — is a curated library of prebuilt agent *samples/templates* (RAG, code modernization, financial analysis, invoice processing) with GitHub source, exportable into the Agent Development Kit; deployable via Agents CLI to Agent Runtime or registered into Gemini Enterprise. Notably it **still carries Pre-GA Offerings Terms** ("as is", limited support) per the [official docs](https://docs.cloud.google.com/gemini-enterprise-agent-platform/build/agent-garden) — a year-plus after its Cloud Next '25 debut, the template tier has not been elevated to a supported GA product.
- **Agent Marketplace / Agent Gallery** — the deployable tier. Launched with [Gemini Enterprise](https://cloud.google.com/blog/products/ai-machine-learning/introducing-gemini-enterprise) (Oct 2025, [$30/seat/mo; Gemini Business $21](https://www.cnbc.com/2025/10/09/google-launches-gemini-enterprise-to-boost-ai-agent-use-at-work.html)): [partner-built agents](https://cloud.google.com/blog/products/ai-machine-learning/partner-built-agents-available-in-gemini-enterprise) from Adobe, Atlassian, Oracle, Salesforce, ServiceNow, Workday, Accenture, Deloitte et al., "pre-vetted… validated to integrate," surfaced in the Agent Gallery in the end-user app. At Next '26 Google added a [$750M partner innovation fund](https://thenextweb.com/news/google-cloud-next-ai-agents-agentic-era) for agent builders.
- **Sovereign angle (watch closely):** Gemini is **GA on Google Distributed Cloud air-gapped** — authorized for US Government Secret and Top Secret, runnable on a single air-gapped server — with Agentspace search in preview on GDC ([Google blog, Aug 2025](https://cloud.google.com/blog/products/ai-machine-learning/run-gemini-and-ai-on-prem-with-google-distributed-cloud); [VentureBeat](https://venturebeat.com/technology/googles-gemini-can-now-run-on-a-single-air-gapped-server-and-vanish-when-you-pull-the-plug)). The *agent platform and marketplace* remain cloud-only, but the direction is unmistakable.

## AWS — AgentCore + a 900-listing marketplace on day one

AWS went marketplace-first and at scale: the [AI Agents and Tools category in AWS Marketplace](https://aws.amazon.com/about-aws/whats-new/2025/07/ai-agents-tools-aws-marketplace/) launched **July 16, 2025** with **900+ listings** (Anthropic, IBM, Salesforce, PwC, Stripe, Perplexity, C3.ai…), natural-language search, MCP and A2A protocol support, and one-path deployment into [Bedrock AgentCore](https://aws.amazon.com/blogs/aws/introducing-amazon-bedrock-agentcore-securely-deploy-and-operate-ai-agents-at-any-scale/) Runtime or AgentCore Gateway ([TechTarget, Jul 2025](https://www.techtarget.com/searchenterpriseai/news/366627853/AWS-launches-AgentCore-system-and-agentic-marketplace)). AgentCore itself (framework- and model-agnostic runtime, gateway, memory, identity, observability) went **GA October 13, 2025** with VPC/PrivateLink support. Pricing is pure consumption — [Runtime $0.0895/vCPU-hr + $0.00945/GB-hr, Gateway per 1K API calls](https://aws.amazon.com/bedrock/agentcore/pricing/) — and AWS is pushing **outcome-based pricing** for ISV agents via Marketplace ([APN blog](https://aws.amazon.com/blogs/apn/agentic-saas-your-next-growth-market-is-already-here/)); AgentCore Payments entered preview May 11, 2026 (agentic commerce signal). Curation is classic AWS Marketplace: listing review + ISV-supported, not AWS-supported. **No on-prem/air-gapped story found for AgentCore.**

## Microsoft — Agent Store for users, Agent 365 registry for admins, Foundry for builders

Microsoft has the cleanest three-way split, and it monetized the *registry*, not the catalog:

- **Foundry** (rebranded Azure AI Foundry): [Agent Service](https://learn.microsoft.com/en-us/azure/foundry/agents/overview) with a **GA tool catalog** and preview "skills" — versioned in a project-scoped catalog, discoverable as MCP resources ([Build 2026 blog](https://devblogs.microsoft.com/foundry/agent-service-build2026/)). Publishing agents to Teams/M365 Copilot hit GA around June 2026. Foundry-native agents carry no platform charge — you pay tokens/tools ([pricing](https://azure.microsoft.com/en-us/pricing/details/foundry-agent-service/)).
- **M365 Copilot Agent Store**: end users find/add agents inside the Copilot app; admins review each agent's capabilities, data access, and permissions and approve/reject from a single workflow ([Microsoft Learn](https://learn.microsoft.com/en-us/microsoft-365/admin/manage/manage-copilot-agents-integrated-apps?view=o365-worldwide)).
- **Agent 365** — the control plane/registry for *all* agents regardless of where they were built — went **GA May 1, 2026 at $15/user/month** (one user license covers all agents that user owns/uses/sponsors; bundled in M365 E7 at $99/user/mo) ([Microsoft Security blog](https://www.microsoft.com/en-us/security/blog/2026/05/01/microsoft-agent-365-now-generally-available-expands-capabilities-and-integrations/); [SAMexpert licensing analysis](https://samexpert.com/agent-365/)). Its [Agent Registry](https://learn.microsoft.com/en-us/microsoft-365/admin/manage/agent-registry?view=o365-worldwide) in the M365 admin center is explicitly pitched as the answer to agent sprawl. This is the closest philosophical cousin to our MCP Gateway/Registry + MLflow Agent Registry combination — and Microsoft is charging per-seat for it.

## Databricks — Unity Catalog as the agent registry, harness pluralism as strategy

At DAIS 2026 (June 16, 2026) Databricks declared Agent Bricks a full agent platform, claiming **100K+ agents built** on it ([Agent Bricks DAIS 2026 blog](https://www.databricks.com/blog/agent-bricks-dais-2026)). What matters for us:

- **GA**: [Supervisor Agent](https://www.databricks.com/blog/agent-bricks-supervisor-agent-now-ga-orchestrate-enterprise-agents) (managed orchestration), **Unity Catalog registry for agents, tools, and models** (agents governed like data assets), the **Omnigent managed meta-harness**, Document Intelligence, and Lakebase-backed agent memory.
- **Preview**: **Unity AI Gateway** — catalog discovery of all agents/models/MCPs including *externally hosted* ones, fine-grained access control, per-user/group budget enforcement, policy-based routing — plus MCP support in Unity Catalog.
- **Harness posture**: Agent Bricks natively runs third-party harnesses — LangGraph, CrewAI, Agno, **Claude Code SDK, OpenAI Agent SDK** — alongside its proprietary managed Omnigent. The message: don't license a harness, host them all and own the governance layer.
- **MLflow angle (directly relevant to our 3.6 EA2 registry):** [MLflow 3.0](https://www.databricks.com/blog/mlflow-30-unified-ai-experimentation-observability-and-governance) (Jun 2025) added agent tracing/observability, a prompt registry, and monitoring for agents deployed **anywhere, including on-prem**. Our MLflow-based Agent Registry builds on Databricks-led OSS — good for velocity and standards alignment, but Databricks steers the roadmap and sells the managed version. Deployment is serverless-only (Databricks Apps); **no on-prem**.

## IBM — the closest analog, and it runs on our platform

IBM's watsonx Orchestrate **[Agent Catalog](https://www.ibm.com/products/watsonx-orchestrate/agent-catalog) has been GA since Think 2025 (May 2025)** with **150+ prebuilt, deployable agents and tools** from IBM and "trusted partners" (Box, Mastercard, Oracle, Salesforce, ServiceNow, Symplistic.ai, 11x), onboardable from any framework, any LLM, any cloud, with observability dashboards and audit trails. (Verification note: a secondary source misdated this to Think 2026 — IBM's Think 2026 announcement, May 5, 2026, was the *next generation* of Orchestrate in **private preview**.) Two things make IBM the most instructive comparison:

1. **It's the only major vendor shipping a GA agent catalog with a true on-prem/air-gapped path** — watsonx Orchestrate [installs on Red Hat OpenShift via IBM Software Hub](https://www.ibm.com/docs/en/watsonx/watson-orchestrate/base?topic=started-installing-watsonx-orchestrate-premises), with [documented air-gapped operation](https://developer.watson-orchestrate.ibm.com/tutorials/tutorial_5_operating_in_an_airgapped_environment) via mirrored private registries; some configurations require Red Hat OpenShift AI underneath.
2. **Coopetition:** IBM monetizes the agent/catalog layer *above* our platform layer. Every on-prem Orchestrate deal validates the "agents on OpenShift" thesis — and also occupies the business-buyer relationship RHOAI's catalog could otherwise grow into. The RHOAI Agent Catalog needs a crisp story about where it complements Orchestrate (platform-engineer/builder audience, open harnesses, no IBM stack required) versus competes.

## NVIDIA — blueprints, not a marketplace; the support-tier model closest to ours

NVIDIA's [Blueprints](https://build.nvidia.com/blueprints) (launched as [NIM Agent Blueprints, Aug 2024](https://nvidianews.nvidia.com/news/nvidia-and-global-partners-launch-nim-agent-blueprints-for-enterprises-to-make-their-own-ai); renamed Oct 2024) are reference agentic workflows — sample app + NIM microservices + reference code + Helm chart — **free to try and download, supported in production only under an NVIDIA AI Enterprise license**, delivered through Accenture, Dell, HPE, Lenovo, Cisco et al. The OSS [NeMo Agent Toolkit](https://github.com/NVIDIA/NeMo-Agent-Toolkit) is deliberately framework-agnostic (LangChain, CrewAI, Semantic Kernel, Google ADK) with MCP support, and the [Enterprise AI Factory validated design](https://docs.nvidia.com/ai-enterprise/planning-resource/ai-factory-white-paper/latest/agentic-ai-in-the-factory.html) packages agentic AI for **on-prem** deployment. Two takeaways: (a) NVIDIA's "free templates → paid supported production" is the closest existing analog to RH's planned supported-harness-image tier — the RHEL model applied to agents is legible to the market because NVIDIA already trained buyers on it; (b) as the NVIDIA partner-of-record for open inference (vLLM/llm-d), RHOAI is a natural landing zone for Blueprints on-prem — a catalog that surfaces NVIDIA Blueprints deployable on OpenShift AI would convert the partnership into catalog content.

## Salesforce & ServiceNow — the SaaS-native marketplaces (context, not direct competitors)

- **Salesforce** launched [AgentExchange March 4, 2025](https://www.salesforce.com/news/press-releases/2025/03/04/agentexchange-announcement/) with 200+ partners and security-reviewed actions/topics/templates, then in **April 2026 collapsed AppExchange, the Slack Marketplace, and the Agentforce ecosystem into a single [AgentExchange](https://www.salesforce.com/agentforce/agentexchange/)** — 10,000 apps, 2,600 Slack apps, **1,000+ Agentforce agents/sub-agents/tools/MCP servers**, semantic search, a $50M builders fund, and the AppExchange brand retired ([Salesforce Ben](https://www.salesforceben.com/appexchange-slack-marketplace-and-the-agentforce-ecosystem-are-now-one-with-fresh-50m-funding/); [SalesforceDevops.net, Apr 14, 2026](https://salesforcedevops.net/index.php/2026/04/14/agentexchange-salesforces-bet-that-trust-can-scale-with-agentic-speed/)). The signal: the 20-year app-marketplace playbook is being rebuilt agent-first, with **mandatory security review** as the trust mechanism.
- **ServiceNow** runs an [AI Agent Marketplace inside the ServiceNow Store](https://store.servicenow.com/store/ai-marketplace) and in **January 2026** rebuilt its [partner program](https://newsroom.servicenow.com/press-releases/details/2026/ServiceNow-enhances-global-Partner-Program-to-accelerate-AI-agent-innovation/default.aspx) around partner-built agents — simplified single membership fee, new tiers, an in-store recommendation agent — while noting [$1B in AWS Marketplace transactions](https://newsroom.servicenow.com/press-releases/details/2026/ServiceNow-hits-1-billion-in-AWS-Marketplace-transactions-as-enterprises-rapidly-adopt-AI-at-scale/default.aspx) (agents increasingly transact through *other vendors'* marketplaces too).

## How competitors handle the debates we're having

- **Template-vs-deployable split:** now the established pattern, not a dilemma. Google splits Agent Garden (samples, pre-GA) from Agent Marketplace (vetted deployable); Microsoft splits Foundry samples/tool catalog from the end-user Agent Store; IBM and Salesforce lead with deployable. RHOAI 3.5's link-out-only template catalog matches Agent Garden circa early 2025; the credibility test is how fast 3.6 makes entries deployable. Notably, even Google has kept its template tier in preview indefinitely — the *deployable, governed* tier is where GA investment goes.
- **Supported/certified tiers:** every marketplace has a trust mechanism — Salesforce mandatory security review, Google "pre-vetted/validated," ServiceNow store certification + partner tiers, IBM "trusted partner" curation, Microsoft admin approval workflow. But **nobody ships vendor-*supported* runnable harness images** — the market offers managed runtimes (AgentCore, Agent Engine, Databricks Apps) instead. RH's "Red Hat-maintained supported harness image" is genuinely differentiated: it is the only model that gives an on-prem customer a supported artifact they run themselves. NVIDIA's free-blueprint/paid-AI-Enterprise-support is the nearest precedent and shows the model sells.
- **Proprietary harness licensing:** the market says don't. Google ADK and Microsoft Agent Framework are open source; Databricks *hosts* proprietary harnesses (Claude Code SDK, OpenAI Agent SDK) rather than licensing its own, and its proprietary Omnigent is a managed service, not a licensed artifact. Value is accruing to the **registry/governance/runtime layer**, not the harness. This supports keeping RH harness images open/upstream-based with support as the product.
- **Air-gapped/sovereign:** still rare, but no longer empty. Who can actually do it: **IBM** (Orchestrate on OpenShift, documented air-gapped installs), **Google** (Gemini GA on GDC air-gapped, Secret/TS authorized — but no agent marketplace there yet), **NVIDIA** (Enterprise AI Factory on-prem validated designs), **HPE** ([Sovereign AI Factory with air-gapped operation, Jun 2026](https://www.forbes.com/sites/maribellopez/2026/06/24/hpe-updates-hardware-private-cloud-and-networking-for--agentic-ai-era/)). AWS, Azure (agent stack), Databricks, Salesforce, ServiceNow: no air-gapped agent-platform story found in this run. The disconnected *agent catalog + registry + gateway* as one governed, self-managed stack remains an open slot — but the window is narrowing.

## Analyst view 2025–2026

- **Gartner** is the loudest skeptic: [over 40% of agentic AI projects will be canceled by end of 2027](https://www.gartner.com/en/newsroom/press-releases/2025-06-25-gartner-predicts-over-40-percent-of-agentic-ai-projects-will-be-canceled-by-end-of-2027) (Jun 2025), and it coined **"agent washing"** — estimating only ~130 of thousands of self-described agentic vendors are real. It also projects 40% of enterprise apps will embed task-specific agents by end-2026 (vs <5% in 2025). There is **no dedicated agent-platform/marketplace MQ yet**; agent coverage fragments across the first [MQ for Enterprise AI Coding Agents (May 20, 2026)](https://www.gartner.com/en/documents/7879277), the 2026 Conversational AI Platforms MQ, AI Application Development Platforms (Google a Leader), and a first [AI Governance Platform MQ (Jul 2026)](https://sanjmo.medium.com/inside-gartners-first-ai-governance-platform-magic-quadrant-fa1f182f75e9). No Red Hat placement in any agent-adjacent MQ found.
- **Forrester** ([State of Agentic AI in 2026, Jun 3, 2026](https://www.forrester.com/blogs/the-state-of-agentic-ai-in-2026-companies-are-chasing-few-are-catching/)): three-quarters of enterprises *claim* agentic adoption, only a small minority run it in meaningful production; **over half report agent sprawl**; the "trust tax" of audit-defensible logging is a core blocker. Its prescriptions — invest in orchestration infrastructure before scaling, treat agents as governed identities with credentials and lifecycle ownership — read like a spec for MCP Gateway/Registry + agent identity. Forrester launched an [Agentic Development Platforms Vendor Landscape (Q3 2026, 25 vendors)](https://www.forrester.com/blogs/launching-the-agentic-development-platforms-vendor-landscape-q3-2026/) with a full **Wave expected Q4 2026** — a near-term inclusion target for RH.
- **IDC** sizes the prize: agentic AI to exceed [26% of worldwide IT spending and $1.3T by 2029](https://my.idc.com/getdoc.jsp?containerId=prUS53765225), already 10–15% of enterprise IT spend in 2026; [FutureScape 2026](https://www.idc.com/resource-center/blog/futurescape-2026-moving-into-the-agentic-future/) forecasts 45% of organizations orchestrating agents at scale by 2030.
- **Red Hat mentions** come from second-tier firms: Futurum covered Red Hat Summit 2026 favorably ("[builders and operators at the center of agentic AI](https://futurumgroup.com/insights/red-hat-brings-developers-product-and-operations-to-the-center-of-agentic-ai/)" — AgentOps, agent identity, skills repository), and [Techaisle](https://techaisle.com/blog/654-red-hat-ai-platform-play-from-any-app-to-any-model-any-hardware-any-cloud) framed the "any model, any hardware, any cloud" play. Tier-1 analyst narratives about agent platforms do not yet feature Red Hat.

## Positioning assessment — honest read

**Where the RHOAI Agent Catalog has a defensible wedge:**

1. **The self-managed, disconnected, governed agent stack.** No hyperscaler offers its agent catalog + registry + gateway as something a customer runs on their own metal. IBM does — on OpenShift. Google's air-gapped Gemini proves demand at the highest classification levels but doesn't yet include the agent platform. A RHOAI catalog whose entries deploy air-gapped, with MCP Gateway policy and MLflow registry lineage intact, is a claim with exactly one other credible maker (IBM, whose stack requires IBM software above ours).
2. **Support model, not marketplace scale.** "Red Hat-maintained supported harness images" is the one curation model nobody else ships (NVIDIA's AI Enterprise support tier is the proven analog). Against Gartner's agent-washing backlash and 40%-cancellation forecast, "a small catalog of things Red Hat actually supports in production" is a stronger enterprise message than "900 listings."
3. **Governance-first matches analyst-defined buying criteria.** Forrester says governance/observability decide; Microsoft charging $15/user for a registry proves willingness to pay. RH AI's public AgentOps (tracing, cryptographic agent identity, eval hub, MLflow) plus MCP Gateway/Registry is the right center of gravity — infra-priced, which is a TCO wedge against per-seat ($30 Gemini Enterprise, $15 Agent 365) and per-call (AgentCore) meters at fleet scale.
4. **Neutrality.** Every hyperscaler catalog funnels to its models, its runtime, its commerce. Databricks is the only other credibly harness-plural platform, and it's serverless-only. Open harnesses + any model + any cloud + on-prem is a combination only RH offers end to end.

**Where we are behind — no varnish:**

1. **Timeline.** Agent Garden shipped with ADK at Cloud Next '25; AWS launched with 900+ listings in July 2025; IBM's 150-agent catalog has been GA since May 2025 and Microsoft's Agent 365 registry hit GA in May 2026. RHOAI 3.5's catalog arrives ~12–18 months after the template tier was table stakes, and the MLflow Agent Registry (3.6 EA2) lands roughly a year after Unity Catalog's agent registry and Agent 365 went GA. The internal urgency framing ("everyone has a registry now") is directionally correct.
2. **Link-out-only at launch is below the 2026 bar.** Every competitor catalog either deploys in-place or is one click from it. If 3.5 ships link-out templates without a visible, dated path to 3.6 deployability, it will demo poorly against Agent Garden — which is *also* templates but exports directly into a managed runtime.
3. **No ecosystem flywheel.** Google put $750M and Salesforce $50M behind partner agent builders; AWS/Salesforce run commerce (including outcome-based pricing) through their marketplaces. An RH-curated-only catalog has no ISV pipeline, no certification program, no monetization for partners — the mechanism that made hyperscaler catalogs grow 10x in a year.
4. **Distribution to end users.** Competitors put agents where users already are (M365 Copilot, Gemini app, Slack, ServiceNow). The RHOAI catalog reaches platform engineers and data scientists; without a consumption surface or a partner like IBM Orchestrate in front, the business buyer never sees it.
5. **Analyst invisibility.** RH appears in no tier-1 agent-platform evaluation. The Forrester ADP Wave (Q4 2026) is the nearest actionable target; missing it cedes the "agentic platform" category narrative for another cycle.
6. **Managed-runtime gap.** AgentCore, Agent Engine, and Databricks Apps give developers serverless agent hosting in minutes. "Bring your own OpenShift cluster" is the honest trade for sovereignty, but it's friction the wedge messaging must own rather than hide.

**Net:** the wedge is real but narrow and time-boxed — *open, portable, supportable, governable, disconnected* — and it must be claimed loudly before (a) IBM's OpenShift-based catalog becomes the default answer to "on-prem agents" and (b) Google extends its air-gapped GDC story from models to the agent platform.
