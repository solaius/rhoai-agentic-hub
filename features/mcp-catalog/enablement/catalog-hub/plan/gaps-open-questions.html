Gaps & Open Questions

The MCP Catalog's two committed-but-unshipped gaps, plus the open questions that touch the Catalog: two raised by this feature's own upstream research, and seven carried from the management hub's cross-cutting open-questions log.

Known Gaps

Gap
Catalog Does Not Create Prerequisites

Issue: Deploying an MCP server from the Catalog requires manually creating a ServiceAccount, RBAC (ClusterRoleBinding), and ConfigMap. The Catalog's "Deploy" button does not automate this today, by design: auto-creating them would require the catalog controller to hold privilege-escalation rights.

Impact: Users must prepare the target namespace with the correct prerequisites before deployment succeeds. The failure mode is silent: the deployment is created, but pods crash.

Owner: RHOAI Catalog Team

Status: Prerequisite automation targeted for RHOAI 3.6 EA1 (RHAIRFE-2434, Approved; delivery Feature RHAISTRAT-2027, In Progress, fix version 3.6 EA1 RHOAI RELEASE per this feature's own Jira sweep; supersedes the obsoleted RHAISTRAT-1938). An open ask, not shipped.

Workaround: Create the ServiceAccount, ClusterRoleBinding, and ConfigMap before deploying from the Catalog; see Enablement & Troubleshooting for the exact commands.

Two older sources date this to RHOAI 3.5 TP: the management hub's plan/gaps.html and its pre-replan plan/roadmap.html. This page follows the Jira-verified 3.6 EA1 dating, consistent with Enablement & Troubleshooting and Roadmap.

Gap
STDIO Transport Not Supported

Issue: The Catalog and Registry currently support Streamable HTTP transport only. STDIO-based MCP servers cannot be listed, registered, or consumed through the governed path (see the Transport requirement on Partner Onboarding Pipeline).

Impact: Local and IDE-native MCP servers, a large share of community and developer tooling, are invisible to Catalog search and cannot be tracked in the Registry.

Owner: RHOAI Catalog Team, RHOAI Registry Team

Status: Committed direction, not shipped. Three RFEs filed 2026-06-26: RHAIRFE-2514 (Catalog: list STDIO MCPs with local install info), RHAIRFE-2515 (Registry: allow registration/tracking of STDIO MCPs), and RHAIRFE-2516 (IDE registry access: install STDIO MCPs / configure remote HTTP-SSE MCPs from IDEs). Delivery Feature RHAISTRAT-2131 (STDIO listings in Catalog) is status New with no fix version yet, confirmed in this feature's own Jira sweep. The paired Registry and IDE-API delivery Features, RHAISTRAT-2132 and RHAISTRAT-2133, are status New with no fix version per the management hub's sweep; they fall outside this feature's own RHAISTRAT scope.

Workaround: None today; STDIO MCP servers must be installed and tracked manually outside the governed Catalog/Registry path.

RHAIRFE-2516 framing corrected to the Jira-sourced title (management hub's plan/jira-tracker.html); the management hub's plan/gaps.html and this feature's own govern/partner-pipeline.html both paraphrase it as "spec-compliant IDE-native registry API."

Open Questions Raised by This Feature

Two questions came out of this feature's own upstream research and are filed as standalone knowledge entries, not rows in the management hub's log:

Open question: official registry federation posture. The Catalog has no stated federation posture toward the protocol-official MCP registry (registry.modelcontextprotocol.io): whether Red Hat implements its subregistry OpenAPI spec (becoming a governed subregistry), polls it as an aggregator only, or stays fully closed. This is a decision for the TP window (RHOAI 3.6 EA1); it shapes TP scope, the metadata schema, and whether Red Hat publishes its own servers to the official registry namespace (none found there today). See Upstream & Ecosystem for the full landscape and question-official-registry-federation.md.

Open question: spec-version compatibility policy at GA. The largest MCP specification revision since launch finalizes 2026-07-28 (streamable HTTP becomes stateless, the initialize handshake and SSE resumability are removed), roughly three months ahead of the Catalog's own GA code freeze (RHOAI 3.6, Oct 23, 2026). No compatibility-matrix policy is stated yet for which spec versions catalog entries declare, what the Gateway accepts, or what the Red Hat and Partner tiers require at GA. See Upstream & Ecosystem and question-spec-version-policy-ga.md.
