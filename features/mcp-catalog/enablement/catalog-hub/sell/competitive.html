Differentiation Read

The claim that "nobody owns enterprise governance end-to-end" is eroding: AWS, Databricks, Docker, and Kong are all building discovery, governance, and runtime together. Red Hat's still-unmatched cards are the shipped single-platform discover-deploy-connect-consume chain, and hybrid/multi-cloud open-source positioning against single-cloud lock-in. UBI-plus-scanning trust is now table stakes, not a differentiator. The three-way Directory/Registry/Enterprise-Platform framing survives as internal analysis, but it is not market vocabulary.

Registry governance is roadmap, not shipped. Where competitors' governance moves (Docker AI Governance, Databricks Contextual Service Policies, AWS approval workflow) invite a comparison to Red Hat's MCP Registry, the honest comparison point is a design-phase product: Registry Dev Preview is not landing in RHOAI 3.5 stable, Tech Preview is targeted for RHOAI 3.6 EA1, and its current data model has no approval, verification, or certification status fields. The four-track governance model (Lifecycle, Approval, Verification, Certification) is a deferred proposal for a post-MVP follow-on. Competing today on governance would overstate what has shipped.

Gap List

They have, we lack (stated publicly)

  • Composable signed catalog artifacts plus team profile remixing (Docker)
  • Content-aware per-action policy: blocking a tool call by request content pending approval (Databricks Contextual Service Policies)
  • Natural-language/semantic search over a private org catalog plus an audit-backed approval workflow (AWS)
  • Spec-aligned OAuth 2.1 Resource Server posture, stated loudly (Kong)
  • Multi-source federation into named catalogs with per-catalog JWT visibility (ToolHive, Red-Hat-co-maintained, already running on OpenShift; a channel-conflict risk with MCPLO/Catalog that needs a stated story)
  • SaaS-provider-hosted remote MCP servers as a supply tier (Salesforce/MuleSoft); Red Hat's tiers currently assume deployable images

We have, they lack

  • A shipped, single-platform discover-deploy-connect-consume chain
  • Hybrid/multi-cloud, open-source positioning against single-cloud lock-in (AWS, Google, Databricks, Salesforce)
  • Full AI-platform bundling (inference, llm-d, NVIDIA) under the same subscription as the catalog
