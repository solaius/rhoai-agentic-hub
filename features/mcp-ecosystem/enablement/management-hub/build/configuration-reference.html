Configuration Reference

Quick reference for all configmaps, CRDs, labels, and configuration artifacts used in the MCP ecosystem.

Configuration Artifacts

Artifact Namespace Purpose Format
odhdashboardconfigs patch redhat-ods-applications Enable MCP Catalog and Gen AI Studio features mcpCatalog: true, genAiStudio: true
gen-ai-aa-mcp-servers ConfigMap redhat-ods-applications Register MCP servers for Gen AI Studio consumption JSON: {"url": "...", "description": "..."}
opendatahub.io/dashboard: "true" Target namespace label Make namespace visible in RHOAI UI project selector Namespace metadata label (must apply via manifest)
ServiceAccount + ClusterRoleBinding MCP server namespace RBAC for MCP server (e.g., cluster-wide read) Standard Kubernetes RBAC
ConfigMap (server config) MCP server namespace MCP server runtime configuration Per-server format (e.g., TOML for OpenShift MCP)
MCPServer CRD MCP server namespace Deploy MCP server via MCPLO mcp.x-k8s.io/v1alpha1
MCPGatewayExtension CRD Gateway namespace Configure MCP Gateway Kuadrant/mcp-gateway
AuthPolicy CRD Gateway namespace Authentication and authorization Kuadrant Authorino

Full YAML Examples

Enable MCP Catalog

oc patch odhdashboardconfigs odh-dashboard-config \
    -n redhat-ods-applications \
    --type=merge \
    -p '{"spec":{"dashboardConfig":{"mcpCatalog":true}}}'

Namespace with Dashboard Label

apiVersion: v1
kind: Namespace
metadata:
  labels:
    opendatahub.io/dashboard: "true"
  name: my-mcp

gen-ai-aa-mcp-servers ConfigMap

apiVersion: v1
kind: ConfigMap
metadata:
  name: gen-ai-aa-mcp-servers
  namespace: redhat-ods-applications
data:
  OpenShift-MCP: |
    {
      "url": "http://ocp-mcp.my-mcp.svc.cluster.local:8080/mcp",
      "description": "Red Hat OpenShift MCP server"
    }
  GitHub-MCP: |
    {
      "url": "http://github-mcp.my-mcp.svc.cluster.local:8080/mcp",
      "description": "GitHub MCP server"
    }
Gateway Integration

When using MCP Gateway, put the gateway URL in this configmap instead of individual server URLs. The Gateway will handle routing to backend servers based on identity and VirtualMCPServer configuration.

data:
  MCP-Gateway: |
    {
      "url": "https://gateway.apps.example.com/mcp",
      "description": "MCP Gateway (all servers)"
    }

ServiceAccount and RBAC

# Service account
apiVersion: v1
kind: ServiceAccount
metadata:
  name: mcp-viewer
  namespace: my-mcp

---
# ClusterRoleBinding for cluster-wide read access
apiVersion: rbac.authorization.k8s.io/v1
kind: ClusterRoleBinding
metadata:
  name: my-mcpviewer-read
roleRef:
  apiGroup: rbac.authorization.k8s.io
  kind: ClusterRole
  name: view
subjects:
- kind: ServiceAccount
  name: mcp-viewer
  namespace: my-mcp

MCP Server ConfigMap (OpenShift MCP Example)

apiVersion: v1
kind: ConfigMap
metadata:
  name: openshift-mcp-server-config
  namespace: my-mcp
data:
  config.toml: |
    log_level = 5
    port = "8080"
    read_only = true
    toolsets = ["core", "config"]

MCPServer CRD

apiVersion: mcp.x-k8s.io/v1alpha1
kind: MCPServer
metadata:
  name: ocp-mcp
  namespace: my-mcp
spec:
  image: quay.io/rhoai-mcp/openshift-mcp-server:latest
  serviceAccountName: mcp-viewer
  configMapRef:
    name: openshift-mcp-server-config
  ports:
  - name: mcp
    containerPort: 8080
    protocol: TCP
  env:
  - name: LOG_LEVEL
    value: "debug"

MCPGatewayExtension CRD

apiVersion: kuadrant.io/v1alpha1
kind: MCPGatewayExtension
metadata:
  name: mcp-gateway
  namespace: mcp-gateway
spec:
  gatewayRef:
    name: istio-gateway
    namespace: gateway-system
  trustedHeadersKey: wristband-public-key
  virtualServers:
  - name: default
    servers:
    - name: openshift-mcp
      url: http://ocp-mcp.my-mcp.svc.cluster.local:8080/mcp

AuthPolicy CRD (Gateway-Level)

apiVersion: kuadrant.io/v1beta2
kind: AuthPolicy
metadata:
  name: mcp-gateway-auth
  namespace: mcp-gateway
spec:
  targetRef:
    group: gateway.networking.k8s.io
    kind: Gateway
    name: istio-gateway
    namespace: gateway-system
  rules:
    authentication:
      "keycloak-jwt":
        jwt:
          issuerUrl: https://keycloak.example.com/realms/mcp
    response:
      success:
        headers:
          "x-authorized-tools":
            opa:
              rego: |
                package wristband
                # Extract per-server tool roles from JWT resource_access claim
                # Sign into ES256 wristband JWT
                # (Full Rego policy omitted for brevity)
          "x-mcp-virtualserver":
            json:
              properties:
                "value":
                  selector: auth.identity.groups

HTTPRoute for MCP Server (with Credential Injection)

apiVersion: gateway.networking.k8s.io/v1
kind: HTTPRoute
metadata:
  name: github-mcp-route
  namespace: mcp-gateway
spec:
  parentRefs:
  - name: istio-gateway
    namespace: gateway-system
  rules:
  - matches:
    - path:
        type: PathPrefix
        value: /mcp/github
    backendRefs:
    - name: github-mcp
      port: 8080

DataScienceCluster Configuration

Ensure RHOAI has Llama Stack Operator managed:

apiVersion: datasciencecluster.opendatahub.io/v1
kind: DataScienceCluster
metadata:
  name: default
spec:
  components:
    llamastackoperator:
      managementState: Managed
    dashboard:
      managementState: Managed

Kuadrant Configuration

apiVersion: kuadrant.io/v1beta1
kind: Kuadrant
metadata:
  name: kuadrant
  namespace: gateway-system

ReferenceGrant (Cross-Namespace)

Allow MCPGatewayExtension in mcp-gateway namespace to reference Gateway in gateway-system:

apiVersion: gateway.networking.k8s.io/v1beta1
kind: ReferenceGrant
metadata:
  name: allow-mcp-to-gateway
  namespace: gateway-system
spec:
  from:
  - group: kuadrant.io
    kind: MCPGatewayExtension
    namespace: mcp-gateway
  to:
  - group: gateway.networking.k8s.io
    kind: Gateway
