Component Directory

One card per component of the MCP Management ecosystem - what it does, where it is in its release path, and where to go for depth. Each component now has its own knowledge hub; this directory is the front door.

How to read this directory

The four MCP Management components compose into one governed path: Catalog discovers, Registry governs, the Lifecycle Operator deploys, and the Gateway enforces at runtime. For a needs-first view, see Which Component Do I Need? Follow each card's hub link for CRDs, install steps, and roadmaps.

MCP Gateway

MCP Gateway
v0.7.1 TP

The runtime enforcement layer: an API gateway for AI tool calls. Provides virtual MCP servers (aggregating tools from multiple backends), identity-aware tool filtering, per-tool authentication, metrics, and audit logging. Built on Red Hat Connectivity Link (a WASM plugin for Envoy, not a standalone gateway), so it adds minimal disruption to existing infrastructure and ships with the RHOAI subscription for AI workloads.

Status: Technical Preview since RHCL 1.3.3 (April 30, 2026); current RHCL 1.4, latest release v0.7.1 (June 18, 2026). GA targeted for RHCL 1.5 (October 2026) with MCP Registry integration in scope.

MCP Catalog

MCP Catalog
3.4 DP

The discovery layer: browse partner, community, and Red Hat MCP servers with metadata - tool lists, security indicators (SAST, verified source, read-only tools), license (SPDX), provider logo, and README. The Catalog is where a user finds a server and clicks Deploy, which hands off to the Lifecycle Operator.

Status: Developer Preview shipped in RHOAI 3.4 (code freeze April 10, 2026), continuing through 3.5 with enhancements. Technical Preview targeted for RHOAI 3.6 EA1 with Registry integration (PM direction, Jul 2026); GA targeted for RHOAI 3.6 (code freeze Oct 23, 2026).

MCP Lifecycle Operator

MCP Lifecycle Operator (MCPLO)
v0.2.0 DP

The deployment primitive underneath the rest of the ecosystem: a Kubernetes-native operator that provides the MCPServer CRD and creates Deployment, Service, and NetworkPolicy from a declarative spec. It ships under the standard OpenShift entitlement, not an RHOAI-restricted one.

Status: DP shipped as v0.2.0 (June 17, 2026). Two qualification tracks converge at GA: OCP-side functional TP targets mid-July 2026 (OCPSTRAT-3263), RHOAI-side deployment TP targets mid-August 2026 (RHAISTRAT-1773). GA targeted for RHOAI 3.6 (code freeze Oct 23, 2026; RHAISTRAT-1995, OCPSTRAT-2879).

MCP Registry

MCP Registry
In Design

The authoritative governance layer and system of record for MCP servers, from discovery through approval, deployment, and consumption. Its current data model is two enums (MCPServerStatus, MCPPublishState) plus an update mode - not the four-track governance model from earlier drafts, which remains a deferred proposal.

Status: Design phase, no code shipped. Dev Preview will not land in RHOAI 3.5 stable. Tech Preview targeted for RHOAI 3.6 EA1 (owner ruling, 2026-07-11), alongside Catalog TP; GA targeted for RHOAI 3.6 (RHAISTRAT-1993, code freeze Oct 23, 2026).

AI Gateway
Adjacent initiative

Not one of the four MCP Management components, but the sibling control plane they compose with. AI Gateway must serve four traffic types - inference, tool calling (MCP), agent-to-agent (A2A), and external provider egress - and its group-based tenancy model for MCP tool catalogs overlaps with the MCP Registry's governance model. Included here as a pointer so the boundary is clear: MCP traffic runs through the MCP Gateway; AI Gateway is the broader enterprise AI control plane discussed at the AI Gateway F2F.
