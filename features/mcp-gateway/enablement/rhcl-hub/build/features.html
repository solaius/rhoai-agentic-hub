Features & Capabilities

MCP Gateway provides enterprise-grade governance, security, and operational features for Model Context Protocol servers. Features are delivered across three milestones: Shipped, Technical Preview, and Planned for GA.

Shipped Features

The following features are available in MCP Gateway v0.7.1 (June 2026) and RHCL 1.3.3+:

Virtual MCP Servers
Shipped

Aggregate tools from multiple backend MCP servers into a single unified view. Agents see one combined tool list instead of managing connections to multiple backends.

Why it matters: Reduces agent complexity, simplifies tool discovery, and enables centralized governance over distributed MCP backends.

Identity-Aware Tool Filtering
Shipped

Filter which tools are visible to which users or agents based on identity. Define allow/deny rules per tool, user, or group via Authorino AuthConfig.

Why it matters: Enforces least-privilege access — users only see tools they're authorized to use, preventing accidental or malicious misuse of sensitive capabilities.

Per-Tool Authentication
Shipped

Apply different authentication policies to different tools. Require OAuth for sensitive operations (e.g., Jira issue creation) while allowing anonymous access for read-only tools.

Why it matters: Fine-grained auth matches the security posture of each tool's risk profile, avoiding "all or nothing" access control.

Per-Tool Authorization
Shipped

Define authorization policies per tool using Authorino. Support for role-based access control (RBAC), attribute-based access control (ABAC), and custom policy expressions.

Why it matters: Enables "user X can call github/create_pr but not github/delete_repo" policies without backend changes.

Per-Tool Metrics
Shipped

Automatic Prometheus metrics for every tool invocation: latency, success/error rates, request counts by tool name and user. Integrated with OpenShift monitoring stack.

Why it matters: Visibility into which tools are used, by whom, and how often — critical for capacity planning, debugging, and compliance audits.

Streaming HTTP Proxy
Shipped

Support for Server-Sent Events (SSE) streaming of MCP responses. Long-running tool calls stream results incrementally rather than blocking until completion.

Why it matters: Enables real-time user feedback for slow operations (e.g., generating a report, searching large datasets) without timeouts.

OAuth 2.0 Protected Resources
Shipped

Full OAuth 2.0 authorization code flow and token exchange (RFC 8693) support. Integrate with Keycloak for identity-aware delegation chains (user → agent → tool).

Why it matters: Enterprise-grade security with audit trails showing who authorized what action, even when mediated through an AI agent.

Technical Preview Features

The following features are available in Technical Preview as of RHCL 1.3.3 (April 2026) and are expected to reach GA in Q4 2026:

Content Guardrails
Tech Preview

Integrate NeMo Guardrails or Red Hat TrustyAI via Envoy ext_proc adapter to scan and filter tool inputs/outputs. Detect PII, profanity, prompt injection, jailbreak attempts, and policy violations before they reach backends.

Why it matters: Centralized content safety without modifying every backend MCP server — enforce governance at the gateway layer.

MCP Elicitation
Tech Preview

Support for the MCP elicitation protocol extension, allowing tools to request additional context or confirmation from the agent before execution.

Why it matters: Enables interactive workflows (e.g., "which Jira project?" when multiple match) without hardcoding assumptions in the tool.

Vault Integration
Tech Preview

Automatic injection of secrets from HashiCorp Vault or External Secrets Operator into tool call contexts. Backend MCP servers retrieve credentials via gateway-provided context instead of direct Vault access.

Why it matters: Simplifies secret management for MCP server developers — no need to bundle Vault SDKs or manage lease renewal logic.

Router OTEL Tracing
Tech Preview

OpenTelemetry trace log emission for every MCP request. Distributed tracing shows the full path: agent → gateway → router → backend → response, with timing breakdowns.

Why it matters: Essential for debugging performance bottlenecks and understanding end-to-end latency in multi-hop agentic workflows.

Isolated Gateway Deployment
Tech Preview

Deploy multiple independent MCP Gateway instances in the same cluster, each with its own virtual MCP server configuration and auth policies. Enable multi-tenancy with hard namespace boundaries.

Why it matters: Critical for shared platforms serving multiple teams or customers with isolated tool sets and compliance requirements.

Hosts for Virtual MCP Servers
Tech Preview

Assign dedicated hostnames to virtual MCP servers (e.g., internal-tools.mcp.example.com, customer-tools.mcp.example.com). Simplifies routing and DNS management for multi-gateway deployments.

Why it matters: Makes it easier to present different tool catalogs to different agent populations without complex path-based routing rules.

Planned for GA (Q4 2026)

The following features are planned for General Availability in Q4 2026:

Deployment Operator
Planned

Kubernetes Operator for automated MCP Gateway lifecycle management. One-click installation, rolling upgrades, automated cert renewal, and configuration drift detection.

Why it matters: Eliminates manual Helm chart management and reduces operational overhead for platform teams.

Security Plugin
Planned

DDoS protection, bot management, and anomaly detection plugin. Detect and block suspicious traffic patterns (e.g., rapid tool enumeration, credential stuffing) before they reach backends.

Why it matters: Protects against both malicious actors and misbehaving agents that could overwhelm backend services.

Session Management
Planned

Resumable MCP sessions with server-side state persistence. Agents can disconnect and reconnect without losing context (e.g., multi-step workflows spanning hours or days).

Why it matters: Enables long-running agentic workflows that survive network interruptions or agent restarts.

High Availability
Planned

Active health checks for backend MCP servers, automatic failover to healthy replicas, and circuit breaker patterns to prevent cascading failures.

Why it matters: Production-grade reliability — agents continue working even when individual backend replicas fail.

Registry Integration
Planned

Automatic configuration from MCP Registry metadata. Register an MCP server in the registry → gateway discovers and configures it automatically without manual CRD creation.

Why it matters: Closes the governance loop — registry approval drives gateway configuration, eliminating manual sync between systems.

Extended MCP Capabilities
Planned

Full MCP protocol support beyond tools: prompts (retrieve prompt templates from backends), resources (attach context documents), sampling (LLM-backed tool execution with model orchestration).

Why it matters: Unlocks advanced agentic patterns — agents can not only call tools but also query for optimal prompts and retrieve relevant documents through the same gateway.
