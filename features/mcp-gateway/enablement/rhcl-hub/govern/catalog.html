MCP Catalog

The storefront for MCP discovery, from the Gateway's vantage point.

Purpose

The Catalog is the discovery layer for MCP servers: partner, community, and Red Hat-provided servers with metadata (tool lists, security indicators, license, provider logo, README), selectable for deployment through the Lifecycle Operator. It shipped as Developer Preview in RHOAI 3.4 (code freeze April 10, 2026) with 10 servers across the three tiers, and is targeted for Technical Preview in RHOAI 3.6 EA1 alongside Registry integration.

Relationship: The Catalog builds on the Registry (authoritative source), complements the Gateway (runtime enforcement), and operates in a federated and interoperable ecosystem.
Planned scope (PM vision, not shipped): The Catalog's long-term vision extends beyond MCP servers to manage Tools, Resources, Prompts, and Workloads, including workload-to-tool mappings, as first-class catalog entities. Planned capabilities include Tool Masking, Input Simplification, and Output Filtering (splitting multi-purpose tools into single-purpose variants and trimming LLM-irrelevant response fields), plus federation with external registries: Docker MCP Catalog, GitHub MCP Registry, Smithery, Microsoft Copilot Catalog, and Databricks Unity Catalog. None of this is shipped; current scope is MCP server discovery, browsing, and selection.
