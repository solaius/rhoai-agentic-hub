MCP Lifecycle Operator

The deployment primitive for MCP server workloads, from the Gateway's vantage point.

Repository: kubernetes-sigs/mcp-lifecycle-operator
Current Version: v0.2.0 (released June 17, 2026)

Purpose

The Lifecycle Operator is the deployment layer: a Kubernetes operator that creates Deployment, Service, and NetworkPolicy from MCPServer CRD definitions, deploying the MCP services the Gateway later routes traffic to. The Registry represents the governed MCP and its deployment/runtime associations; the operator does NOT own governance or lifecycle state, and does not itself route Gateway traffic.

Independence from Gateway: Standalone from MCP Gateway — lifecycle operator installs first, gateway is downstream. No dependency on MCP Gateway for deployment.
Entitlement (corrected): MCPLO ships under the standard OpenShift entitlement, not an RHOAI-restricted one: base OCP customers get MCPLO's deployment capability without RHOAI, and can upgrade to RHOAI later without losing MCPLO functionality. RHOAI layers the MCP Registry, Authorino, and Limitador on top; none of the three is a prerequisite for MCPLO itself.

This replaces an earlier claim on this page naming an RHOAI-restricted subscription tier as the minimum requirement for MCPLO, which was stale. Source: MCP Lifecycle Operator Knowledge Hub: Distribution & Entitlement (Slack architecture decision, mrunalp, Feb 2026).

Full depth: MCP Lifecycle Operator Knowledge Hub: architecture, the three-repo integration pattern, installation, the MCPServer CRD reference, qualification, security, distribution, entitlement, and roadmap.
