MCPServer CRD Reference

The MCPServer custom resource (mcp.x-k8s.io/v1alpha1) is MCP Lifecycle Operator (MCPLO)'s single deployment primitive. Its current schema, verified against the upstream CRD 2026-07-10, nests configuration under source, config, and runtime. It is not a flat spec.

Full Example

apiVersion: mcp.x-k8s.io/v1alpha1
kind: MCPServer
metadata:
  name: ocp-mcp
  namespace: my-mcp
spec:
  source:
    type: ContainerImage
    containerImage:
      ref: registry.redhat.io/openshift-mcp-beta/openshift-mcp-server-rhel9:0.2
  config:
    port: 8080
    path: /mcp
    arguments:
    - --config
    - /etc/mcp-config/config.toml
    storage:
    - path: /etc/mcp-config
      permissions: ReadOnly
      source:
        configMap:
          name: openshift-mcp-server-config
  runtime:
    security:
      serviceAccountName: mcp-viewer

Verbatim from the MCP Management Knowledge Hub's Configuration Reference page, verified against the upstream CRD 2026-07-10.

Field Reference

Field Type / Values Purpose
spec.source.type ContainerImage Selects the source type for the MCP server workload. Currently the only supported value.
spec.source.containerImage.ref OCI image reference The container image to run, e.g. registry.redhat.io/openshift-mcp-beta/openshift-mcp-server-rhel9:0.2.
spec.config.port integer Container port the MCP server listens on.
spec.config.path string HTTP path for the MCP endpoint, e.g. /mcp.
spec.config.arguments list of strings Command-line arguments passed to the server process.
spec.config.storage list of objects (path, permissions, source) ConfigMap or Secret mounts. The operator validates referenced ConfigMap/Secret existence before rollout.
spec.runtime.security.serviceAccountName string RBAC scoping for the workload. The operator cannot auto-create ServiceAccounts, a deliberate security decision; users bring their own SA with appropriate RBAC bindings. Defaults to the namespace's default SA if omitted.

Auto-Created Resources

All auto-created resources share the MCPServer's name:

  • Deployment: security-hardened by default (non-root, read-only rootfs, all capabilities dropped, seccomp RuntimeDefault)
  • Service: cluster-internal discovery
  • NetworkPolicy: ingress-only on the configured port (reconciled automatically since upstream v0.2.0)
  • Pods: labeled mcp-server=<name>

Status Conditions & Discovery

Status field Meaning
Accepted (reason: Valid) Config validation passed. If a referenced ConfigMap/Secret does not exist, this is set to False with a clear error message instead of creating a broken deployment.
Ready (reason: Available) Deployment is available and the MCP protocol handshake succeeded.
status.deploymentName, status.serviceName Names of the auto-created Deployment and Service.
status.address.url Cluster-internal MCP endpoint, pattern http://<name>.<namespace>.svc.cluster.local:<port>/mcp. Still http://, not https://, today; see Qualification & Security for the in-cluster TLS gap.

kubectl get mcpservers shows: READY, ACCEPTED, IMAGE, PORT, ADDRESS.

Validation Behavior

  • Missing ConfigMap/Secret references produce a permanent Accepted=False validation error (no requeue); other, transient errors get exponential backoff.
  • Injects a default TCP socket readiness probe if none is specified.
  • Performs an MCP protocol-level handshake beyond the TCP health check: once the Deployment is available, connects via StreamableClientTransport (10s timeout), sends an Initialize request, and extracts server info and capabilities. Auth errors (401/403) count as reachable.

Prerequisites

Kubernetes v1.28+.
