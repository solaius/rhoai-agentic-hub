Quick Reference

Essential facts for the MCP Lifecycle Operator (MCPLO): the three-repo architecture, the MCPServer CRD, the release train, key Jiras, and entitlement.

Upstream & Architecture Repositories

Repo Role
kubernetes-sigs/mcp-lifecycle-operator Upstream SIG Apps operator; reconciles MCPServer CRs (the deployment primitive)
opendatahub-io/mcp-lifecycle-module-operator ODH module wrapper; reconciles MCPLifecycleOperator CRs and deploys the upstream controller as an operand
opendatahub-io/opendatahub-operator Parent operator; embeds the module handler in the DataScienceCluster (DSC) CR as the mcplifecycleoperator component

Downstream fork github.com/openshift/mcp-lifecycle-operator adds CI/build infrastructure but core operator code tracks upstream.

MCPServer CRD Quick Facts

Field Value
API group mcp.x-k8s.io/v1alpha1
Kind MCPServer
Prerequisite Kubernetes v1.28+
Auto-created resources Deployment, Service, NetworkPolicy (security-hardened: non-root, read-only rootfs, dropped capabilities, seccomp RuntimeDefault)

Milestone Owner / qualification Target Jira Status

The OCP-side TP lands earlier than the RHOAI-side TP because OCP owns functional qualification while RHOAI owns deployment/integration qualification: two separate tracks converging at GA.

Entitlement

MCPLO ships with OCP entitlement

MCPLO is not RHOAI-restricted. Base OpenShift customers get MCPLO with a standard OCP entitlement; RHOAI adds Registry, Authorino, and Limitador on top. Customers can start on OCP and upgrade to RHOAI later without losing MCPLO functionality.
