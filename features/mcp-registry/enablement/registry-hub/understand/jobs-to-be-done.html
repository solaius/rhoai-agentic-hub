The real jobs the MCP Registry serves, sourced from the hub's narrative layer: the cross-feature jobs-to-be-done inventory that ties product surfaces back to what a persona is actually trying to accomplish.

As of this writing, one narrative job-to-be-done names mcp-registry in its features list (confirmed by grep -l "mcp-registry" narrative/knowledge/jtbd-*.md on 2026-07-12). More jobs may attach to the Registry over time as the narrative layer grows; this page reflects the current set.

Discover my skills, MCP servers, and agentic assets

Persona: ai-engineer   Status: candidate

When I'm building or configuring agents,
I want to discover governed skills, MCP servers, and agentic assets,
so I can compose capable agents from validated, approved components rather than unvetted tools.

How the Registry addresses it:

  • The job's own "How RHOAI addresses this" list names the MCP Registry specifically as the governance component: version, lifecycle, and policy for MCP servers, distinct from the MCP Catalog's storefront role
  • The Registry is the reason a discovered server can be trusted, not just found: it supplies the identity, version, and lifecycle state (MCPServerStatus, MCPPublishState) that a discovery surface can key off, once the two are wired together
  • Workspace-scoped, RBAC-aware visibility means an engineer sees the servers approved for their workspace, not an unfiltered global list, once the Registry ships
  • The certification context this job implies (knowing a component is "validated, approved") maps most directly onto the four-track proposal's verification and certification tracks, which are a deferred proposal, not part of the current two-enum model; see Data Model

This job is served conceptually today, not through live cross-component wiring: the Registry has not shipped, and Registry-to-Catalog surfacing integration is its own tracked Tech Preview item (RHAISTRAT-2027), targeted for RHOAI 3.6 EA1 alongside the Registry's own Tech Preview.

Pillar: Agents (AI Hub theme + MCP Ecosystem theme). Evidence status: not yet backed by direct customer evidence (tracked as "candidate" in the narrative inventory).
