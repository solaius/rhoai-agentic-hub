Raised as an open question for the upstream MLflow proposal: which MCP server metadata fields should be mandatory on a registry record vs. optional/best-effort? Relevant to both the proposed data model ([fact-mcp-registry-data-model-proposal.md](/features/mcp-registry/knowledge/fact-mcp-registry-data-model-proposal.md)) and the catalog metadata schema ([fact-mcp-catalog-metadata-schema.md](/features/mcp-ecosystem/knowledge/fact-mcp-catalog-metadata-schema.md)), which notes no standardized upstream schema exists yet — all 4 partner MCPs in the 3.4 catalog shipped with different metadata shapes.
