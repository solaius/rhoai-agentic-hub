**Context**: Draft messaging (including an early AgentOps Topics Page draft) referred to a "dedicated AgentOps control plane," raised in Kagenti discussion and Agentic Messaging Guide review comments.

**Decision**: Rejected — OpenShift IS the control plane. Red Hat is extending OpenShift with agentic needs, not building a separate control plane (Catherine Weeks clarification).

**Consequences**: Messaging materials using "AgentOps control plane" need correction. As of source capture, the AgentOps Topics Page draft still contained this exact contradiction and had not yet been fixed — check current draft status before reusing that copy.

Note: source gives no exact clarification date (undated review comments); `decided` set to drafting date.
