Enablement & Troubleshooting

How to turn the MCP Catalog on in RHOAI, and the two most common failure points when deploying an MCP server from it. Organized as Symptom, Cause, Fix.

Enable the Catalog

The Catalog is enabled with a patch to the dashboard config, the same patch used to flip on the enablement flag documented in the management hub's Operator Installation and Configuration Reference pages:

oc patch odhdashboardconfigs odh-dashboard-config \
    -n redhat-ods-applications \
    --type=merge \
    -p '{"spec":{"dashboardConfig":{"mcpCatalog":true}}}'

mcpCatalog: true patch carried verbatim from the management hub's Operator Installation and Configuration Reference pages.

Deployment Issues

Known Issue
"failed to create Deployment: ConfigMap not found"

Symptom: Deploying an MCP server from the catalog fails with an error like:

failed to create Deployment: ConfigMap "openshift-mcp-server-config" not found

Cause: Prerequisites not created before deploying from catalog. The catalog does NOT automatically create ServiceAccounts, ConfigMaps, or RBAC resources (this would require privilege escalation).

Fix: Create prerequisites BEFORE deploying the MCP server from the catalog:

oc create serviceaccount mcp-viewer -n my-mcp
oc create clusterrolebinding my-mcpviewer-read \
    --clusterrole view \
    --serviceaccount my-mcp:mcp-viewer

oc apply -f - <<EOF
apiVersion: v1
kind: ConfigMap
metadata:
  name: openshift-mcp-server-config
  namespace: my-mcp
data:
  config.toml: |
    log_level = 5
    port = "8080"
    read_only = true
    toolsets = ["core", "config"]
EOF
Known Issue
MCP Server Not Appearing in Catalog Deployments

Symptom: After deploying an MCP server from the catalog, it doesn't appear in AI Hub → MCP Servers → Deployments.

Cause: One of the following:

  • Namespace missing opendatahub.io/dashboard: "true" label
  • Server image not found or inaccessible
  • MCPServer CR not created successfully (check oc get mcpserver -n my-mcp)

Fix: Verify namespace label and MCPServer CR status:

oc get namespace my-mcp --show-labels
oc get mcpserver -n my-mcp
oc describe mcpserver ocp-mcp -n my-mcp

Check the MCPServer status for errors. If image pull fails, verify the image reference and registry access.

Prerequisite Automation: RHAIRFE-2434

Current limitation. Neither issue above has an automated fix today: the catalog does not create ServiceAccount, ConfigMap, or ClusterRoleBinding prerequisites for you, by design (auto-creating them would require the catalog controller to hold privilege-escalation rights). Prerequisite automation is tracked under RHAIRFE-2434 (Approved; delivery Feature RHAISTRAT-2027, In Progress; supersedes the obsoleted RHAISTRAT-1938), targeted for RHOAI 3.6 EA1 as part of Catalog/Registry Integration TP: an open ask, not shipped.

Two management-hub pages date this differently: plan/gaps.html reads "planned for 3.5 TP," while the Jira-sourced plan/open-questions.html and plan/jira-tracker.html both tie RHAIRFE-2434 to RHOAI 3.6 EA1 with an explicit delivery Feature (RHAISTRAT-2027). This page follows the more specific, Jira-verified 3.6 EA1 dating.
