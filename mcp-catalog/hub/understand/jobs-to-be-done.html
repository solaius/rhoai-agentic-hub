Discover my skills, MCP servers, and agentic assets

Persona: ai-engineer   Status: candidate

When I'm building or configuring agents,
I want to discover governed skills, MCP servers, and agentic assets,
so I can compose capable agents from validated, approved components rather than unvetted tools.

How the Catalog addresses it:

  • The Catalog is the storefront for Red Hat, Partner, Community, and (planned) approved enterprise MCP servers in one browsable UI, with trust tier display so an engineer can judge provenance before pulling a server into an agent
  • The partner certification pipeline (consent, technical attestation, image build) plus per-tool security indicators (verifiedSource, sast, secureEndpoint, readOnlyTools) give discovery a validated floor, not just an open list
  • The Catalog sits at the front of the discover, deploy, connect, consume chain, handing off to the Lifecycle Operator and Gateway so a discovered server becomes a governed, connected capability rather than a dead-end listing
  • Multi-registry federation (planned) widens what's discoverable beyond Red Hat's own curation to Docker MCP Catalog, GitHub MCP Registry, and partner catalogs

Pillar: Agents (AI Hub theme + MCP Ecosystem theme). Evidence status: not yet backed by direct customer evidence (tracked as "candidate" in the narrative inventory).
