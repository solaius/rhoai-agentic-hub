Operator Installation

The MCP ecosystem on RHOAI involves a two-tier operator installation model. Installation order matters — Tier 1 must complete before Tier 2.

Two-Tier Model

The ecosystem is built on two layers of infrastructure:

Tier Components Installation Method Purpose
Tier 1
Platform Prerequisites		 • OpenShift Service Mesh
• cert-manager
• Red Hat Connectivity Link
• RHOAI 		OperatorHub Platform-level infrastructure for API management, TLS, and AI workloads
Tier 2
MCP Ecosystem		 • MCP Gateway Operator
• MCP Lifecycle Operator
• RHBK (Keycloak)
• HashiCorp Vault 		OLM custom catalog, Manual YAML, Helm MCP-specific management, auth, and credential injection
Installation Order Matters

All Tier 1 operators must be installed and healthy before installing Tier 2 components. MCP Gateway requires Service Mesh and Kuadrant to be fully operational.

Tier 1: Platform Prerequisites

1. OpenShift Service Mesh 3.x

Provides Envoy-based ingress gateway and ext_proc support required by MCP Gateway Router.

oc apply -f - <<EOF
apiVersion: operators.coreos.com/v1alpha1
kind: Subscription
metadata:
  name: servicemeshoperator
  namespace: openshift-operators
spec:
  channel: stable
  name: servicemeshoperator
  source: redhat-operators
  sourceNamespace: openshift-marketplace
EOF

2. cert-manager

TLS certificate management and ClusterIssuers for Gateway TLS termination.

oc apply -f - <<EOF
apiVersion: operators.coreos.com/v1alpha1
kind: Subscription
metadata:
  name: openshift-cert-manager-operator
  namespace: cert-manager-operator
spec:
  channel: stable-v1
  name: openshift-cert-manager-operator
  source: redhat-operators
  sourceNamespace: openshift-marketplace
EOF

3. Red Hat Connectivity Link (RHCL)

Kuadrant platform providing Authorino (auth), Limitador (rate limiting), and DNSPolicy/TLSPolicy CRDs.

oc apply -f - <<EOF
apiVersion: operators.coreos.com/v1alpha1
kind: Subscription
metadata:
  name: kuadrant-operator
  namespace: openshift-operators
spec:
  channel: stable
  name: kuadrant-operator
  source: redhat-operators
  sourceNamespace: openshift-marketplace
EOF

After installation, apply a Kuadrant CR to activate the control plane:

oc apply -f - <<EOF
apiVersion: kuadrant.io/v1beta1
kind: Kuadrant
metadata:
  name: kuadrant
  namespace: gateway-system
EOF

4. Red Hat OpenShift AI (RHOAI)

May require a custom CatalogSource for RHOAI 3.4.0+ (check with your Red Hat account team for access).

After installation, ensure the DataScienceCluster has llamastackoperator: Managed:

oc patch datasciencecluster default \
    --type=merge \
    -p '{"spec":{"components":{"llamastackoperator":{"managementState":"Managed"}}}}'

Tier 2: MCP Ecosystem

1. MCP Lifecycle Operator

Not available via OLM or Helm. Install manually from the GitHub release asset (repo is kubernetes-sigs/mcp-lifecycle-operator; openshift/mcp-lifecycle-operator is a fork with no releases of its own):

oc apply -f https://github.com/kubernetes-sigs/mcp-lifecycle-operator/releases/download/v0.2.0/install.yaml

# Wait for operator to be ready
oc wait --for=condition=Ready pod -l control-plane=controller-manager \
    -n mcp-lifecycle-operator-system --timeout=300s
Since v0.2.0 (upstream)

The operator now reconciles a NetworkPolicy per MCPServer operand automatically, so no manual NetworkPolicy prerequisite is needed. Its metrics/webhook server also supports TLS security-profile tuning via the TLS_MIN_VERSION and TLS_CIPHER_SUITES environment variables.

2. MCP Gateway Operator (for full Gateway setup)

Install via OLM using a custom CatalogSource:

oc apply -f - <<EOF
apiVersion: operators.coreos.com/v1alpha1
kind: CatalogSource
metadata:
  name: mcp-gateway-catalog
  namespace: openshift-marketplace
spec:
  sourceType: grpc
  image: quay.io/kuadrant/mcp-gateway-operator-catalog:v0.7.1
  displayName: MCP Gateway Catalog
  publisher: Kuadrant
EOF

# Subscribe to the operator
oc apply -f - <<EOF
apiVersion: operators.coreos.com/v1alpha1
kind: Subscription
metadata:
  name: mcp-gateway-operator
  namespace: openshift-operators
spec:
  channel: alpha
  name: mcp-gateway-operator
  source: mcp-gateway-catalog
  sourceNamespace: openshift-marketplace
EOF

3. Red Hat Build of Keycloak (RHBK)

For identity management and OIDC authentication:

oc apply -f - <<EOF
apiVersion: operators.coreos.com/v1alpha1
kind: Subscription
metadata:
  name: rhbk-operator
  namespace: openshift-operators
spec:
  channel: stable-v24
  name: rhbk-operator
  source: redhat-operators
  sourceNamespace: openshift-marketplace
EOF

4. HashiCorp Vault (Optional)

For dynamic credential injection. Install via Helm:

helm repo add hashicorp https://helm.releases.hashicorp.com
helm repo update

helm install vault hashicorp/vault \
  --namespace vault \
  --create-namespace \
  --set server.ha.enabled=true \
  --set server.ha.replicas=3

Validated Component Versions

These versions have been tested and validated together on ROSA/OpenShift 4.21.6 with RHOAI 3.4.0. Validated as of the RHOAI 3.4 validation run (2026-04-21): this matrix reflects that run, not current-release validation; re-validate before relying on it against a newer RHOAI train.

Component Version Notes
Platform ROSA, OpenShift 4.21.6, RHOAI 3.4.0 Baseline platform
OpenShift Service Mesh 3.3.2 Envoy gateway, mTLS
Red Hat Connectivity Link 1.3.2 (Authorino 1.3.0, Limitador 1.3.0) Includes MCP Gateway 0.6.0 TP
MCP Gateway v0.6.0 RHCL 1.3.3 bundle
MCP Lifecycle Operator release-0.1 Manual install only
Keycloak 26.4.11-opr.1 (RHBK) Identity management
Vault 1.21.2 (Helm 0.32.0) Optional, credential injection
Version Support Gap — August 2026

RHOAI 3.5 (Aug 2026) supports OCP 4.19, but RHCL 1.4+ will likely require OCP 4.20+. The configuration (RHOAI 3.5 + RHCL + OCP 4.19) is untested and best-effort support only.

OCP 4.19 maintenance support ends December 2026 (only 4 months overlap). Customers are advised to upgrade to OCP 4.20+ for full support.

Post-Install Activation

Enable MCP Catalog in RHOAI

oc patch odhdashboardconfigs odh-dashboard-config \
    -n redhat-ods-applications \
    --type=merge \
    -p '{"spec":{"dashboardConfig":{"mcpCatalog":true}}}'

Enable Gen AI Studio (if not already enabled)

oc patch odhdashboardconfigs odh-dashboard-config \
    -n redhat-ods-applications \
    --type=merge \
    -p '{"spec":{"dashboardConfig":{"genAiStudio":true}}}'

Verification

Verify all operators are healthy before proceeding:

# Tier 1
oc get csv -n openshift-operators | grep -E "(servicemesh|cert-manager|kuadrant)"
oc get kuadrant -n gateway-system

# Tier 2
oc get pods -n mcp-lifecycle-operator-system
oc get csv -n openshift-operators | grep mcp-gateway
oc get keycloak -A
oc get pods -n vault  # if Vault installed
