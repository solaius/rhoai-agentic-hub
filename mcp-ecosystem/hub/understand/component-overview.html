Deep dive into each component of the MCP Management ecosystem — status, versions, key CRDs, and upstream repos.

MCP Gateway

MCP Gateway
v0.7.1 TP

Purpose: API gateway for AI tool calls. Provides virtual MCP servers, identity-aware tool filtering, per-tool authentication, metrics, and audit logging.

Status: Technical Preview (TP) since RHCL 1.3.3 (April 30, 2026); current RHCL 1.4. Latest release v0.7.1 (June 18, 2026). GA targeted for RHCL 1.5 (October 2026) with MCP Registry integration.

Key CRDs:

  • MCPServerRegistration (mcp.kuadrant.io/v1): registers an MCP server with the gateway (spec.prefix, targetRef to HTTPRoute)
  • MCPVirtualServer (mcp.kuadrant.io/v1): curated virtual server composed from one or more registered backends
  • MCPGatewayExtension — Configures gateway-level behavior (trustedHeadersKey, broker settings)

Upstream Repo: Kuadrant/mcp-gateway

Platform: Built on Red Hat Connectivity Link (RHCL). RHCL is a WASM plugin for Envoy (not a standalone gateway), meaning minimal disruption to existing infrastructure.

RHOAI Integration: Ships as part of RHOAI subscription (auth via Authorino, rate limiting for AI workloads only). Full RHCL purchase required for DNS, HA/DR/GLB.

Roadmap Highlights:

  • TP (Apr 2026): Isolated Gateway Deployment, Vault Integration, MCP Elicitation, Content Guardrails, OTEL Tracing
  • GA (RHCL 1.5, Oct 2026): Deployment Operator, Security Plugins (DDoS, Bot Management), Session Management, HA with health checks, MCP Registry integration

Learn More: MCP Gateway Knowledge Hub

MCP Catalog

MCP Catalog
3.4 DP

Purpose: Discovery layer for MCP servers. Browse partner, community, and Red Hat MCP servers with metadata (tool lists, security indicators, license, provider logo, README).

Status: Developer Preview (DP) shipped in RHOAI 3.4 (code freeze April 10, 2026); DP continues through 3.5 with enhancements. Technical Preview (TP) targeted for RHOAI 3.6 EA1 with Registry integration (PM direction, Jul 2026). General Availability (GA) targeted for RHOAI 3.6 (code freeze Oct 23, 2026).

MCP Lifecycle Operator

MCP Lifecycle Operator (MCPLO)
v0.2.0 DP

Purpose: Kubernetes-native operator providing the MCPServer CRD, the deployment primitive underneath the rest of the MCP ecosystem, creating Deployment, Service, and NetworkPolicy from a declarative spec.

Status: DP shipped as v0.2.0 (June 17, 2026). Two qualification tracks converge at GA: OCP-side functional TP targets mid-July 2026 (OCPSTRAT-3263), RHOAI-side deployment TP targets mid-August 2026 (RHAISTRAT-1773, on-risk: engineering's bug week falls Jul 13, tracked in OCPMCP-298). GA targeted for RHOAI 3.6 (code freeze Oct 23, 2026; RHAISTRAT-1995, OCPSTRAT-2879). Ships under the standard OpenShift entitlement, not an RHOAI-restricted one.

MCP Registry

MCP Registry
Design Phase

Purpose: Authoritative governance layer and system of record for MCP servers, from discovery through approval, deployment, and consumption. Current data model: two enums (MCPServerStatus, MCPPublishState) plus an update mode, not the four-track model from earlier drafts, which remains a deferred proposal.

Status: Design phase, no code shipped. Dev Preview will not land in RHOAI 3.5 stable. Tech Preview targeted for RHOAI 3.6 EA1 (owner ruling, 2026-07-11), alongside Catalog TP. GA targeted for RHOAI 3.6 (RHAISTRAT-1993, code freeze Oct 23, 2026).
