How this aggregation was built

Every entry under narrative/knowledge/jtbd-*.md carries a features frontmatter list. This page unions the jobs whose list includes any of the five component tags above (checked 2026-07-12; ten jobs exist in total in the narrative layer). Two unique jobs match: one names mcp-gateway, one names both mcp-catalog and mcp-registry (so it appears in both of those groups, once per matching component - the same pattern the component hubs' own Jira rollups use for shared umbrella items). No job in the current inventory names mcp-ecosystem or mcp-lifecycle-operator directly; those two groups are gaps, not empty by omission, and are called out below rather than left silently blank.

MCP Gateway

Govern agent tool and model access

Persona: agentops-admin   Status: candidate

When I manage a fleet of agents, I want to govern which tools and models each agent can access, so I can enforce least-privilege, rate-limit token usage per team, and audit every agent action.

MCP Gateway is named as the tool-access layer of a three-layer governance model (model access, tool access, identity): tool-level access control via OAuth 2.0 token exchange, On-Behalf-Of delegation for least-privilege, and a cryptographic act-claim audit trail.

MCP Catalog

Discover my skills, MCP servers, and agentic assets

Persona: ai-engineer   Status: candidate

When I'm building or configuring agents, I want to discover governed skills, MCP servers, and agentic assets, so I can compose capable agents from validated, approved components rather than unvetted tools.

The Catalog is the storefront for Red Hat, Partner, Community, and (planned) approved enterprise MCP servers, with trust-tier display and a partner certification pipeline giving discovery a validated floor.

MCP Registry

Discover my skills, MCP servers, and agentic assets
Same job as Catalog, above

This job's features list names both mcp-catalog and mcp-registry, so it is one narrative job tabled twice by design, once per matching component. The Registry's angle: it supplies the identity, version, and lifecycle state (MCPServerStatus, MCPPublishState) that makes a discovered server trustworthy, not just found, once Registry-Catalog wiring ships (tracked as RHAISTRAT-2027).
