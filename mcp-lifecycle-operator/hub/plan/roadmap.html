Roadmap

The MCP Lifecycle Operator's (MCPLO) own release plan: Dev Preview shipped, two separate qualification tracks converging at GA, then a forward look to OCP 5.0. The OCP-side track owns functional qualification; the RHOAI-side track owns deployment and integration qualification. They land about a month apart and meet at GA.

The OpenShift/Kubernetes MCP server is tracked separately from MCPLO. MCPLO is the deployment operator; the OpenShift/Kubernetes MCP server it can deploy has its own, distinct Jira tracking: OCPSTRAT-2739 (GA) and OCPSTRAT-3337 (TP2, targeted early Aug 2026). Neither key belongs on this operator's own roadmap table above.

Dependencies

MCPLO delivery hangs off the same umbrella outcome as the rest of the MCP ecosystem, RHAISTRAT-1339 (AI Hub AI Asset Delivery for Agentic Solutions); see Jira Tracker for MCPLO's own tracked slice of that outcome. MCPLO itself has no install-time dependency on the MCP Gateway: MCPLO installs first (or alongside RHOAI) to manage server deployment, while the Gateway is downstream for routing traffic to already-deployed servers. Gateway integration for MCPLO-deployed servers is explicitly scoped to GA, not TP, tracked as OCPMCP-347. The MCP Catalog deploys discovered servers through MCPLO, and the MCP Registry will integrate with it for governance once Registry TP lands; neither of those schedules is MCPLO-owned, so they can move independently of the dates above.
