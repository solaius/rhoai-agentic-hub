MCP Registry Knowledge Hub

The governance backbone underneath the MCP ecosystem on Red Hat OpenShift AI (RHOAI), currently in design phase

The MCP Registry is the system of record for governed MCP servers: identity, version, lifecycle state, certification, trust, and auditability. It fills the gap between the MCP Catalog (discovery), the MCP Lifecycle Operator (deployment), and the MCP Gateway (runtime enforcement) by governing what is approved rather than deploying or enforcing anything itself. It is explicitly not a replacement for any of those components; it is the governance layer underneath them.

This hub is a design-decision record, not a product manual

The MCP Registry is not built. There is no shipped Dev Preview and no code today. This hub exists to capture the design decisions, data model, and governance model as they are made, growing toward Tech Preview at RHOAI 3.6 EA1. Expect pages here to change as the design firms up.
