Roadmap

The MCP Registry's release train from today's design and Dev Preview work through Tech Preview and General Availability, and how it lines up with the MCP Catalog and the Catalog-Registry integration it depends on.

Nothing on this page is shipped

The Registry has no code and no shipped Dev Preview. Every milestone below is a target, not a delivered release.

Dependencies

The Registry is not a standalone delivery. Its own roadmap only moves as fast as three other components let it:

  • MCP Catalog: the Catalog-Registry integration (RHAISTRAT-2027) is the mechanism that lets the Registry's lifecycle state enrich Catalog search results; both reach Tech Preview in the same RHOAI 3.6 EA1 window. See the MCP Catalog Knowledge Hub: Roadmap.
  • MCP Lifecycle Operator (MCPLO): auto-registers deployed servers into the Registry as "discovered" entries (update_mode: SYSTEM_MANAGED); this flow is design-resolved today, independent of MCPLO's own OCP/RHOAI qualification timeline. See the MCP Lifecycle Operator Knowledge Hub: Roadmap.
  • MCP Gateway: the Registry-to-Gateway auto-config flow that replaces today's manual MCPServerRegistration CRD bridge is GA-target work, tracked under RHAISTRAT-1993 alongside RHCL 1.5 (October 2026). See the MCP Gateway Knowledge Hub: Roadmap.

Full flow detail (mechanism, status, and open questions for each of the three integration points) lives on Integrations.
